Anti-patterns

안티패턴은 “하지 말아야 할 것”을 명시해서, 시간이 지나도 디자인이 망가지지 않게 막는 방어막입니다.

Global Anti-patterns

금지 왜 위험한가 대안
페이지마다 다른 버튼 스타일 학습자는 UI를 다시 배워야 함 btn-primary/secondary/ghost만 사용
새 색을 즉흥적으로 추가 의미 체계 붕괴, 유지보수 어려움 토큰 5색으로 해결
텍스트 대비 무시 가독성 하락, 접근성 문제 Ink/Blue로 대비 확보
애니메이션 과다 학습 집중 방해 150–200ms, 목적 있는 모션만

Color Anti-patterns

  • 노랑 배경 위 흰 글자 — 대비 부족
  • 빨강을 기본 강조 — 시험/불안 톤 강화
  • 정답을 색만으로 표시 — 색각 이슈/접근성 문제

Typography Anti-patterns

  • 굵기 난립 — 위계 붕괴
  • 줄간격 축소 — 눈 피로 증가
  • 영어만 다른 폰트 — 시각적 분열

Interaction Anti-patterns

  • 클릭 영역 44px 미만 — 모바일 오입력 증가
  • hover 효과 과장 — 시각 소음
  • 포커스 링 제거 — 키보드/접근성 붕괴
