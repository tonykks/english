UI Patterns

패턴은 “반복되는 문제의 표준 해법”입니다. 기초영어 학습에서는 특히 피드백, 진행, 오류 안내가 핵심입니다.

Pattern 1 — Feedback (정답/오답)

상황 메시지 톤 다음 행동
정답 짧게 칭찬 + 확인 “다음 문장” 버튼 제공
오답 비난 금지, 힌트 제공 “다시 시도” + “힌트 보기”

Pattern 2 — Progress (진행 표시)

진행은 “불안을 줄이는 정보”입니다. 숫자/단계는 과하게 크지 않게, 화면 상단에 조용히 배치합니다.

DoDon’t
“3/10” 처럼 명확한 진행 끝이 안 보이는 로딩/무한 스피너
학습 흐름을 방해하지 않기 진행 표시가 주인공이 되기

Pattern 3 — Errors (오류 안내)

오류 메시지는 “문제”가 아니라 “해결”을 말해야 합니다. (예: “네트워크 오류” ❌ → “인터넷 연결을 확인하고 다시 시도하세요” ⭕)

규칙 설명
원인 추측 최소화 사용자를 탓하지 않기
다음 행동 제공 버튼/링크로 해결 경로 제시
짧고 구체적 한 문장 + 행동 1개
